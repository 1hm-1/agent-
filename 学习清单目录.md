# Agent 开发实习 4 周冲刺学习清单

## 适用目标

- 目标：4 周内开始投递大厂 Agent 开发实习
- 时间预算：每天 8 小时，约 224 小时
- 当前阶段：正在学习 LangChain，需要尽快补齐 Agent 面试高频知识与项目表达

---

## 总原则

1. 先保就业竞争力，再追求知识完备。
2. 先能讲清楚，再追求做复杂系统。
3. 先手搓核心链路，再回头使用框架。
4. 每周都要有可展示成果，不要只学不产出。

---

## 学习主线总览

### 第一优先级

- LLM 基础与 Prompt/上下文工程
- RAG 全链路与评测
- Tool Calling / Function Calling / MCP / Skills 区别
- Agent 基本范式与任务规划
- 手搓一个最小 Agent
- 项目表达与面试问答整理

### 第二优先级

- LangChain 核心抽象与源码级理解
- 多 Agent 框架认知
- OpenClaw / Claude Code / OpenSpec / SDD 等设计哲学
- FastAPI、限流、安全、缓存、基础后端能力

### 第三优先级

- 多模态对齐
- 微调细节
- 大规模分布式工程细节

## 第 1 周：打牢 Agent 核心基础

### 1. LLM 基础补齐

- 学习目标：能解释 token、context window、temperature、top_p、system prompt、few-shot、structured output、hallucination
- 预计时间：10 小时
- 产出要求：整理一份 `LLM基础问答.md`

### 2. Prompt 工程与上下文工程

- 学习目标：理解 system/user/assistant 分层、指令优先级、历史消息裁剪、摘要压缩、上下文拼接策略
- 预计时间：8 小时
- 重点问题：
  - 上下文窗口满了怎么压缩
  - 如何筛选有效信息和垃圾信息
  - Prompt 注入攻击如何防御
- 产出要求：整理一份 `上下文工程设计笔记.md`

### 3. LangChain 核心抽象

- 学习目标：彻底搞懂 `PromptTemplate`、`ChatPromptTemplate`、`Runnable`、`invoke`、`stream`、`output parser`、`MessagesPlaceholder`
- 预计时间：10 小时
- 学习要求：不只会写 demo，要能讲出为什么这样组合
- 产出要求：手写 3 个最小 demo

### 4. 手搓最小单 Agent

- 学习目标：不用 LangChain，直接基于 API 手写一个最小 Agent 循环
- 预计时间：12 小时
- 实现内容：
  - 用户输入
  - 模型决策
  - 工具调用
  - 结果回填
  - 最终回答
- 产出要求：完成第一个可运行项目 `mini_agent`

---

## 第 2 周：RAG 全链路与检索优化

### 5. RAG 标准流程

- 学习目标：掌握数据清洗、切块、向量化、召回、rerank、生成、引用
- 预计时间：10 小时
- 必须能回答：
  - RAG 和微调的区别
  - RAG 的适用场景、优缺点
  - 当前项目为什么选择 RAG 而不是微调

### 6. 检索优化专题

- 学习目标：理解父子索引、BM25、向量检索、混合检索、rerank、topK 截断
- 预计时间：14 小时
- 必须能回答：
  - 为什么引入父子索引
  - 为什么检索阶段要加 BM25
  - rerank 后一般返回几个块
  - topK 截断怎么做
  - 如何提升相关度

### 7. RAG 评测

- 学习目标：理解召回率、相关性、答案正确性、引用准确性、延迟、成本等评测维度
- 预计时间：8 小时
- 必须能回答：
  - RAG 系统如何评测
  - 评测数据集包括什么
  - 如何优化回答效果
- 产出要求：整理 `RAG评测清单.md`

### 8. 完成一个 RAG 项目

- 学习目标：完成一个可讲清楚的文档问答或知识库问答项目
- 预计时间：16 小时
- 项目最低要求：
  - 支持文档导入
  - 支持切块与检索
  - 支持 rerank 或混合检索
  - 支持引用来源
  - 支持失败日志

---

## 第 3 周：Agent 深水区与面试高频专题

### 9. Function Calling / MCP / Skills 对比

- 学习目标：讲清楚三者定义、边界、优缺点、适用场景
- 预计时间：8 小时
- 必须能回答：
  - Function Calling 到 MCP 到 Skills 的演进关系
  - 各自适合解决什么问题
  - 为什么不直接全都用工具调用

### 10. Agent 范式与任务规划

- 学习目标：理解 ReAct、Plan-and-Execute、Router、Workflow、Multi-Agent 等范式
- 预计时间：10 小时
- 必须能回答：
  - Agent 的任务规划怎么做
  - 为什么某些场景不适合用多 Agent
  - 常见 Agent 范式有哪些

### 11. 记忆机制与上下文管理

- 学习目标：理解短期记忆、长期记忆、摘要记忆、向量记忆、结构化记忆
- 预计时间：8 小时
- 必须能回答：
  - 长短期记忆怎么设计
  - 本地记忆存储怎么做
  - 当前上下文是如何处理的

### 12. 安全与工具治理

- 学习目标：理解 Prompt 注入、工具越权、参数校验、白名单、沙箱、人工确认
- 预计时间：8 小时
- 必须能回答：
  - Prompt 注入如何防御
  - 工具调用的安全控制怎么做
  - 高风险工具为什么需要人工确认

### 13. 手搓增强版 Agent

- 学习目标：在第 1 周项目基础上加入任务规划、记忆、工具安全控制
- 预计时间：16 小时
- 项目最低要求：
  - 至少 2 个工具
  - 有上下文管理
  - 有错误重试
  - 有工具调用日志
  - 有简单安全策略

## 第 4 周：补工程能力，准备投递

### 14. FastAPI 与服务化封装

- 学习目标：把 Agent 或 RAG 项目封成可访问接口
- 预计时间：10 小时
- 必学内容：
  - 路由
  - 请求体校验
  - 异常处理
  - 中间件
  - StreamingResponse

### 15. 基础后端八股补齐

- 学习目标：把实习面试高频工程问题补到能答
- 预计时间：14 小时
- 重点专题：
  - 分布式令牌桶限流
  - 漏桶算法
  - 滑动窗口算法
  - 布隆过滤器
  - 数据库索引失效
  - `like` 查询与索引

### 16. 框架与业界方案认知

- 学习目标：补齐 LangChain 之外的框架与设计哲学认知
- 预计时间：10 小时
- 重点专题：
  - Claude Code
  - OpenClaw
  - OpenSpec
  - SDD
  - A2A
  - 常见多 Agent 框架
- 学习要求：不追求源码全懂，但要能讲出设计思路、优缺点、适用场景

### 17. 项目包装与面试素材准备

- 学习目标：把前 3 周成果变成能讲、能投、能答的材料
- 预计时间：14 小时
- 必做产出：
  - 项目 README
  - 项目架构图
  - STAR 法项目介绍稿
  - 高频问题问答清单
  - 简历项目描述

### 18. 模拟面试与补缺

- 学习目标：对照面经查漏补缺
- 预计时间：8 小时
- 方法建议：
  - 每天自问自答 15 题
  - 每题控制在 1 到 3 分钟
  - 重点修正“只会概念，不会落地”的回答

## 面试必答专题清单

### Agent 核心

- 什么是 Agent，和普通 ChatBot 的区别是什么
- Agent 常见范式有哪些
- Agent 的任务规划怎么做
- Function Calling、MCP、Skills 有什么区别
- 多 Agent 什么时候值得引入

### RAG 核心

- RAG 和微调的区别
- 父子索引为什么有用
- 为什么要引入 BM25
- rerank 的作用是什么
- topK 如何截断
- RAG 如何评测
- 如何提升相关度与最终回答质量

### 上下文与记忆

- 上下文窗口满了怎么办
- 短期记忆和长期记忆怎么设计
- 如何压缩历史消息
- 如何避免垃圾信息污染上下文

### 安全与工程

- Prompt 注入怎么防御
- 工具调用安全怎么做
- 限流算法有哪些
- 布隆过滤器解决什么问题
- 数据库索引为什么会失效

### 项目表达

- 为什么做这个项目
- 遇到的最大问题是什么
- 如何优化效果与性能
- 为什么这样设计而不是另一种方案
- 如果重做一次会怎么改
