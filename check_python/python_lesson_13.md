# 第 13 讲：环境变量与项目配置管理

## 这一讲的目标

学完这一讲，你应该能做到：

- 理解什么是环境变量
- 知道为什么密钥不能写死在代码里
- 会读取简单环境变量
- 理解配置和业务代码分离的意义
- 建立基础的项目配置意识

这一讲开始更接近真实工程实践，尤其是你后面接 API 时会立刻用到。

---

## 1. 什么是环境变量？

你可以把环境变量理解成：

- 操作系统提供给程序的一组外部参数
- 程序运行时可以读取它们

例如：

- `OPENAI_API_KEY`
- `OPENAI_BASE_URL`
- `DEBUG`

它们不直接写在 Python 代码里，而是在程序外部提供。

---

## 2. 为什么不能把 API Key 写死？

例如这种写法不推荐：

```python
api_key = "sk-xxxxxxxx"
```

主要问题有：

- 容易泄露
- 提交到代码仓库时风险很大
- 更换密钥不方便
- 不同环境不好切换

所以真实项目里通常会把密钥放在环境变量里。

---

## 3. 如何读取环境变量？

最常用的是 `os.getenv()`。

```python
import os

api_key = os.getenv("OPENAI_API_KEY")
print(api_key)
```

如果环境变量不存在，通常会得到 `None`。

---

## 4. 一个更实用的读取方式

```python
import os

api_key = os.getenv("OPENAI_API_KEY")

if not api_key:
    raise ValueError("缺少 OPENAI_API_KEY 环境变量")
```

这种写法更适合真实项目，因为它能明确提示配置缺失。

---

## 5. 可选配置怎么处理？

有些配置可以没有，比如：

- `OPENAI_BASE_URL`
- `DEBUG`

可以这样写：

```python
base_url = os.getenv("OPENAI_BASE_URL")
debug = os.getenv("DEBUG", "false")
```

这里第二个参数表示默认值。

---

## 6. 什么叫“配置和代码分离”？

意思是：

- 业务逻辑写在代码里
- 可变化的参数放到配置里

例如这些通常属于配置：

- 模型名
- API Key
- base URL
- 是否开启调试
- 最大重试次数

这样做的好处是：

- 更安全
- 更容易维护
- 更容易切换环境

---

## 7. 一个完整例子：读取 Agent 配置

```python
import os

api_key = os.getenv("OPENAI_API_KEY")
model = os.getenv("MODEL_NAME", "gpt-4.1")
debug = os.getenv("DEBUG", "false")

if not api_key:
    raise ValueError("请先设置 OPENAI_API_KEY")

print("model =", model)
print("debug =", debug)
```

注意：

- 这里打印模型和调试开关可以
- 但不要随便打印 API Key 本身

---

## 8. 为什么不要把密钥打印到日志里？

因为日志可能会被：

- 保存到文件
- 发给别人排错
- 上传到服务器

如果密钥出现在日志里，风险会很高。

这是你现在就值得养成的好习惯。

---

## 9. 为什么这对 Agent 开发很重要？

后面你几乎一定会处理：

- 模型配置
- API Key
- 接口地址
- 调试开关

如果没有基本配置意识，项目会很快变乱。

---

## 10. 常见错误

### 错误 1：密钥写死在代码中

这是最常见、也最危险的入门错误之一。

### 错误 2：环境变量缺失时不报清晰错误

程序会莫名其妙失败，不利于排查。

### 错误 3：把密钥打印出来

调试时很容易不小心泄露。

---

## 11. 这一讲你至少要完成的练习

### 练习 1：读取必填环境变量

写一段代码读取 `OPENAI_API_KEY`：

- 如果存在，就打印“已读取到 API Key”
- 如果不存在，就给出清晰报错

### 练习 2：读取可选环境变量

读取 `OPENAI_BASE_URL`，如果没有则返回 `None` 或默认值。

### 练习 3：设计一个简单配置字典

把下面这些配置组织成一个字典：

- 模型名
- 最大重试次数
- 是否开启调试日志

其中一部分可以来自环境变量，一部分可以直接写默认值。

---

## 12. 本讲小结

这一讲最重要的是：

- 环境变量是外部配置来源
- API Key 不应该写死在代码里
- 可变参数应该和业务逻辑分开
- 不要把敏感信息输出到日志里
