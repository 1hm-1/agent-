# 第 8 讲：异常处理与调试基础

## 这一讲的目标

学完这一讲，你应该能做到：

- 理解什么是异常
- 会使用 `try / except`
- 认识几种常见报错
- 学会从报错信息里定位问题
- 建立“调试是编程一部分”的意识

很多人不是不会写代码，而是一报错就卡住。这一讲就是帮你建立基本的排错能力。

---

## 1. 什么是异常？

异常就是程序运行时发生了错误。

例如：

```python
print(1 / 0)
```

这会报错，因为不能除以 0。

异常并不代表你“不适合编程”，它只是程序在告诉你：

- 哪里出问题了
- 什么类型的问题

---

## 2. 常见异常示例

### `ZeroDivisionError`

```python
print(1 / 0)
```

### `IndexError`

```python
numbers = [1, 2, 3]
print(numbers[5])
```

### `KeyError`

```python
user = {"name": "Alice"}
print(user["age"])
```

### `ValueError`

```python
age = int("abc")
```

### `TypeError`

```python
print("18" + 1)
```

---

## 3. `try / except`

有些错误是你预期可能发生的，这时可以捕获它们。

```python
try:
    age = int(input("请输入年龄："))
    print(age)
except ValueError:
    print("你输入的不是有效整数")
```

意思是：

- 先尝试执行 `try` 中的代码
- 如果出现 `ValueError`，就执行 `except`

---

## 4. `finally`

`finally` 中的代码通常无论是否出错都会执行。

```python
try:
    print("开始")
except:
    print("出错")
finally:
    print("结束")
```

当前阶段你只要知道：

- `finally` 常用于清理资源或收尾逻辑

---

## 5. 为什么不要随便写裸 `except`

例如：

```python
try:
    ...
except:
    print("出错了")
```

虽然能用，但不推荐一开始就这么写，因为：

- 你不知道到底是什么错
- 容易把真正的问题藏起来

更推荐尽量写具体异常类型，比如 `ValueError`、`IndexError`。

---

## 6. 怎么看报错信息？

看到报错时，至少关注这几个点：

- 错误类型是什么
- 哪一行出错
- 那一行在做什么

例如：

```python
numbers = [1, 2, 3]
print(numbers[5])
```

如果报 `IndexError`，就说明你在访问不存在的位置。

报错信息不是敌人，它是在给你线索。

---

## 7. 最基础的调试方法：打印

当你不确定某个变量里到底是什么时，先打印出来。

```python
age = input("请输入年龄：")
print(age)
print(type(age))
```

这能帮助你确认：

- 值对不对
- 类型对不对

很多问题用 `print()` 就能快速定位。

---

## 8. 一个完整例子：安全读取年龄

```python
text = input("请输入年龄：")

try:
    age = int(text)
    print("明年你将是：", age + 1)
except ValueError:
    print("请输入有效的整数")
```

这个例子非常接近真实程序思路：

- 先接收输入
- 尝试转换
- 如果失败，给出清晰提示

---

## 9. 为什么异常处理对 Agent 开发很重要？

因为后面你会经常遇到这些问题：

- API 请求失败
- 返回数据结构不符合预期
- 文件不存在
- JSON 解析失败
- 用户输入不合法

如果完全没有异常处理，程序会很脆弱。

---

## 10. 常见错误

### 错误 1：用异常掩盖问题

捕获了异常，但不给出有意义的信息。

### 错误 2：什么都不打印

一出错只看到“失败了”，但不知道为什么。

### 错误 3：把所有错误都当成同一种问题

不同错误通常有不同原因，最好分类处理。

---

## 11. 这一讲你至少要完成的练习

### 练习 1：处理整数输入错误

让用户输入年龄，如果不是整数，就提示重新输入格式不正确。

### 练习 2：处理列表越界

创建一个只有 3 个元素的列表，故意访问不存在的位置，观察错误类型。

### 练习 3：处理字典缺失键

创建一个只包含 `name` 的字典，尝试读取 `age`，观察错误类型，再试试 `get()`。

### 练习 4：处理数字转换

尝试把 `"123"` 和 `"abc"` 分别转成整数，观察不同结果。

### 练习 5：写一个带异常处理的小函数

写函数 `safe_divide(a, b)`：

- 如果 `b` 为 0，返回提示信息
- 否则返回除法结果

---

## 12. 本讲小结

这一讲最重要的是：

- 异常是运行时错误，不是失败证明
- `try / except` 能让程序更稳
- 看懂报错类型和行号，是最基础的调试能力
- `print()` 依然是很有效的调试工具
