# 第 10 讲：文件读写与路径处理

## 这一讲的目标

学完这一讲，你应该能做到：

- 会读取文本文件
- 会写入文本文件
- 理解 `with open(...)` 的基本用法
- 知道为什么要关注编码
- 对路径有基础认识

这一讲非常关键，因为后面做 Agent 时，你会经常把内容写入文件、读取配置、保存日志或对话记录。

---

## 1. 为什么程序需要读写文件？

前面我们写的很多程序，结果都只停留在终端里。

但真实开发常常需要：

- 读取配置文件
- 保存聊天记录
- 写日志
- 读提示词模板

这就需要文件读写。

---

## 2. 打开文件：`open()`

最常见的写法是：

```python
f = open("test.txt", "r", encoding="utf-8")
content = f.read()
f.close()
```

这里：

- `"r"` 表示读取
- `encoding="utf-8"` 表示按 UTF-8 编码读

但现在更推荐用 `with open(...)`。

---

## 3. 推荐写法：`with open(...)`

```python
with open("test.txt", "r", encoding="utf-8") as f:
    content = f.read()

print(content)
```

好处是：

- 用完文件后会自动关闭
- 写法更安全、更清晰

---

## 4. 读取文本文件

假设有文件 `hello.txt`，内容是：

```text
Hello Python
```

读取代码：

```python
with open("hello.txt", "r", encoding="utf-8") as f:
    content = f.read()

print(content)
```

---

## 5. 写入文本文件

```python
with open("output.txt", "w", encoding="utf-8") as f:
    f.write("你好，Python")
```

这里：

- `"w"` 表示写入
- 如果文件不存在，会创建
- 如果文件已存在，通常会覆盖原内容

---

## 6. 追加写入

如果你不想覆盖，而是追加内容，可以用 `"a"`。

```python
with open("log.txt", "a", encoding="utf-8") as f:
    f.write("一条新日志\n")
```

这很适合日志场景。

---

## 7. 为什么要写 `encoding="utf-8"`？

因为文本文件可能包含中文。

如果编码处理不一致，常见问题是：

- 中文乱码
- 读取时报编码错误

所以当前阶段建议你形成习惯：

- 读写文本文件时优先明确写 `encoding="utf-8"`

---

## 8. 路径是什么？

路径就是文件在电脑中的位置。

例如：

```text
D:\项目\agent学习\check_python\python_lesson_10.md
```

这就是一个 Windows 路径。

你当前阶段先理解两点：

- 文件名不等于完整路径
- 程序运行位置不同，相对路径的结果也可能不同

---

## 9. 相对路径和绝对路径

### 相对路径

例如：

```python
open("test.txt", "r", encoding="utf-8")
```

它依赖“当前工作目录”。

### 绝对路径

例如：

```python
open(r"D:\项目\agent学习\check_python\test.txt", "r", encoding="utf-8")
```

它写明了完整位置。

入门阶段你只要知道：

- 相对路径更方便
- 绝对路径更明确

---

## 10. 一个完整例子：保存一句话再读回来

```python
with open("note.txt", "w", encoding="utf-8") as f:
    f.write("我正在学习 Python")

with open("note.txt", "r", encoding="utf-8") as f:
    content = f.read()

print(content)
```

这已经是最基础的“写文件 -> 读文件”流程。

---

## 11. 文件读写时常见问题

### 问题 1：文件不存在

如果你用 `"r"` 去读一个不存在的文件，会报错。

### 问题 2：编码问题

不指定 `utf-8` 时，中文环境下可能出现乱码或错误。

### 问题 3：写入时覆盖原内容

`"w"` 会覆盖文件，要有这个意识。

## 14. 本讲小结

这一讲最重要的是：

- `with open(...)` 是最常用的文件操作方式
