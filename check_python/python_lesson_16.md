# 第 16 讲：异步编程入门与 Agent 项目过渡

## 这一讲的目标

学完这一讲，你应该能做到：

- 理解同步和异步的基本区别
- 知道 `async` 和 `await` 的基本含义
- 能看懂最简单的异步函数
- 理解为什么网络请求场景常常适合异步
- 对一个最小 Agent 项目结构形成整体认识

这一讲不是要求你立刻熟练掌握异步，而是让你“不再害怕看到 `async def`”。

---

## 1. 什么是同步？

同步可以先理解成：

- 一步做完，再做下一步

例如：

1. 发送请求 A
2. 等它返回
3. 再发送请求 B

这种方式简单直接，但有时效率不高。

---

## 2. 什么是异步？

异步可以先理解成：

- 在等待某个操作时，不一定什么都不做

特别是像这些场景：

- 网络请求
- 文件 IO
- 外部服务调用

等待时间可能比较长，这时异步就有价值。

---

## 3. `async def` 是什么？

例如：

```python
async def fake_llm_call():
    return "hello"
```

这表示它是一个异步函数。

异步函数的调用方式和普通函数不完全一样，通常要配合 `await`。

---

## 4. `await` 是什么？

你可以先把它理解成：

- 等待一个异步操作完成，并拿到结果

例如：

```python
import asyncio

async def fake_llm_call():
    await asyncio.sleep(1)
    return "done"
```

这里 `await asyncio.sleep(1)` 表示异步等待 1 秒。

---

## 5. 一个最小异步示例

```python
import asyncio

async def fake_task():
    await asyncio.sleep(1)
    return "task finished"
```

这段代码你当前阶段只要看懂：

- `async def` 定义异步函数
- `await` 等待异步操作

---

## 6. 为什么异步常和 API 调用一起出现？

因为 API 请求通常有等待时间。

例如：

- 等待模型返回
- 等待多个外部工具返回
- 等待多个文件或服务结果

如果完全串行，一个接一个等，会很慢。

异步能帮助程序在等待期间更高效地安排任务。

---

## 7. 你现在需要掌握到什么程度？

当前阶段不要求你立刻熟练写复杂异步代码。

更现实的目标是：

- 看到 `async def` 不慌
- 知道 `await` 大致在做什么
- 能读懂简单示例
- 知道异步常出现在哪些场景

这已经足够支撑你继续往 Agent 开发过渡。

---

## 8. 一个简单的并发示意

```python
import asyncio

async def fake_llm_call(name, delay):
    await asyncio.sleep(delay)
    return f"{name} done"
```

如果你后面同时跑多个类似任务，异步会更有优势。

你现在先不必死磕细节，重点是理解它出现的原因。

---

## 9. 一个最小 Agent 项目大致长什么样？

学到这里，你已经可以理解一个最小项目大致可以这样拆：

```text
agent_project/
  main.py
  config.py
  agent.py
  api_client.py
  utils.py
```

可能分别负责：

- `main.py`：命令行入口
- `config.py`：配置读取
- `agent.py`：消息组织和核心逻辑
- `api_client.py`：请求模型接口
- `utils.py`：通用函数

## 11. 常见错误

### 错误 1：一看到异步就觉得自己不够格

不是。异步本来就是多数人需要反复接触才会熟悉的内容。

### 错误 2：基础还没稳就强行上复杂框架

这样很容易同时卡在：

- Python 语法
- 框架概念
- 调试问题

### 错误 3：忽视项目结构

只会写零散脚本，后面会很难扩展。
