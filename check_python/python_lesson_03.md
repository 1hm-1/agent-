# 第 3 讲：条件判断与流程控制

## 这一讲的目标

学完这一讲，你应该能做到：

- 理解程序为什么需要“判断”
- 会使用 `if / elif / else`
- 理解比较运算符和逻辑运算符
- 能根据不同输入执行不同分支
- 能写出简单的交互式判断程序

这一讲非常关键，因为从这里开始，程序不再只是“顺着往下执行”，而是开始具备“根据情况做不同决定”的能力。

---

## 1. 什么是流程控制？

程序默认是从上到下顺序执行的。

例如：

```python
print("A")
print("B")
print("C")
```

运行结果总是：

```text
A
B
C
```

但真实程序往往不是这样固定的。

比如：

- 如果用户输入的是正确密码，就允许登录
- 如果库存不足，就提示不能购买
- 如果年龄小于 18，就给出不同结果

这时程序就需要“根据条件决定接下来做什么”，这就是流程控制。

---

## 2. 最基本的条件判断：`if`

最简单的写法：

```python
age = 20

if age >= 18:
    print("你已经成年")
```

这里表示：

- 如果 `age >= 18` 这个条件成立
- 就执行缩进里的代码

如果条件不成立，就跳过。

### 结构拆解

```python
if 条件:
    条件成立时执行的代码
```

注意：

- `if` 后面要有条件
- 条件后面要有冒号 `:`
- 下一行要缩进

---

## 3. 如果条件不成立怎么办：`else`

有时候你希望：

- 成立时做一件事
- 不成立时做另一件事

这时就用 `else`。

例如：

```python
age = 16

if age >= 18:
    print("你已经成年")
else:
    print("你还未成年")
```

这里表示：

- 如果年龄大于等于 18，输出成年
- 否则，输出未成年

---

## 4. 多个条件时：`elif`

如果不只是两种情况，而是多种情况，就可以使用 `elif`。

例如：

```python
score = 85

if score >= 90:
    print("优秀")
elif score >= 60:
    print("及格")
else:
    print("不及格")
```

执行顺序是：

1. 先判断 `score >= 90`
2. 如果不成立，再判断 `score >= 60`
3. 如果还不成立，执行 `else`

### 一个重要原则

`if / elif / else` 是按顺序判断的，一旦前面某个条件成立，后面就不会再继续判断。

---

## 5. 比较运算符

条件判断里最常见的是比较运算符。

### 5.1 大于、小于

```python
>
<
>=
<=
```

例如：

```python
age >= 18
score < 60
```

### 5.2 是否相等

```python
==
```

例如：

```python
name == "Alice"
```

注意：

- `=` 是赋值
- `==` 才是比较“是否相等”

这是初学者很容易犯错的地方。

### 5.3 是否不相等

```python
!=
```

例如：

```python
password != "123456"
```

表示“不等于”。

---

## 6. 条件表达式的结果是布尔值

比如：

```python
print(5 > 3)
print(5 == 3)
```

输出会是：

```text
True
False
```

说明：

- 条件成立，结果就是 `True`
- 条件不成立，结果就是 `False`

所以你可以把 `if` 理解成：

- 如果后面的布尔值是 `True`，就执行
- 如果是 `False`，就跳过

---

## 7. 逻辑运算符：`and`、`or`、`not`

有时候一个条件不够，需要把多个条件组合起来。

### 7.1 `and`

表示“并且”，两个条件都为真才为真。

例如：

```python
age = 20
has_ticket = True

if age >= 18 and has_ticket:
    print("可以入场")
```

这里必须同时满足：

- 年龄符合要求
- 有票

### 7.2 `or`

表示“或者”，只要有一个条件为真就为真。

例如：

```python
is_vip = False
has_coupon = True

if is_vip or has_coupon:
    print("可以享受优惠")
```

### 7.3 `not`

表示“取反”。

例如：

```python
is_logged_in = False

if not is_logged_in:
    print("请先登录")
```

---

## 8. 一个完整例子：用户登录判断

```python
username = input("请输入用户名：")
password = input("请输入密码：")

if username == "admin" and password == "123456":
    print("登录成功")
else:
    print("用户名或密码错误")
```

这段代码里你可以看到：

- `input()` 获取输入
- `==` 比较内容是否相等
- `and` 把两个条件合并
- `if / else` 决定不同输出

这已经非常接近真实程序中的判断逻辑了。

---

## 9. 条件判断中的缩进

还是要再次强调：Python 用缩进表示代码块。

例如：

```python
age = 18

if age >= 18:
    print("成年")
    print("可以继续")

print("程序结束")
```

这里：

- 前两条 `print()` 中只有缩进的那部分属于 `if`
- 最后一条没有缩进，不属于 `if`

所以运行逻辑是：

- 如果条件成立，执行缩进内两行
- 然后继续执行最后一行

---

## 10. 常见错误

### 错误 1：把 `=` 写成 `==`

例如：

```python
if age = 18:
    print("ok")
```

这是错误写法，因为在条件判断里你要比较，不是赋值。

正确写法：

```python
if age == 18:
    print("ok")
```

### 错误 2：漏掉冒号

例如：

```python
if age >= 18
    print("成年")
```

会报错，因为 `if` 行末尾必须有冒号。

### 错误 3：缩进不正确

例如：

```python
if age >= 18:
print("成年")
```

这也会报错，因为 `if` 下方必须缩进。

### 错误 4：条件顺序写错

看这个例子：

```python
score = 95

if score >= 60:
    print("及格")
elif score >= 90:
    print("优秀")
```

这里 `score = 95` 时，会先命中 `score >= 60`，所以输出“及格”，后面的“优秀”永远没机会执行。

正确写法应该把更严格的条件放前面：

```python
if score >= 90:
    print("优秀")
elif score >= 60:
    print("及格")
```

## 11. 一个稍微完整一点的小程序：成绩评级

```python
score = int(input("请输入成绩："))

if score >= 90:
    print("优秀")
elif score >= 80:
    print("良好")
elif score >= 60:
    print("及格")
else:
    print("不及格")
```

这个程序综合使用了：

- `input()`
- `int()`
- `if / elif / else`
- 比较运算符

这是很典型的条件分支练习。

---

## 12. 为什么条件判断对 Agent 开发重要？

后面你学 Agent 时，会经常写这种逻辑：

- 如果用户没有输入，就提示重试
- 如果 API 返回错误，就进入异常处理
- 如果需要调用工具，就走工具分支
- 如果结果为空，就返回兜底内容

所以条件判断不是基础里的边角知识，而是程序逻辑本身。

## 14. 本讲小结

这一讲最核心的内容是：

- `if` 用来做条件判断
- `else` 处理不满足条件的情况
- `elif` 用来处理多个分支
- `==` 是比较，`=` 是赋值
- `and`、`or`、`not` 用来组合条件
- 条件判断的结果本质上是布尔值

如果你现在能自己写出“输入年龄后判断是否成年”和“输入成绩后输出等级”这类程序，这一讲就已经掌握得不错了。
