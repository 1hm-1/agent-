# 第 1 讲：重新认识 Python 与开发环境

## 这一讲的目标

学完这一讲，你应该能做到：

- 知道 Python 是什么，适合拿来做什么
- 理解“解释器、脚本、编辑器”分别是什么
- 能在自己电脑上运行一个 Python 程序
- 知道 Python 代码为什么对缩进敏感

## 1. Python 是什么？

Python 是一种通用编程语言，特点通常是：

- 语法相对简洁，容易上手
- 生态丰富，库很多
- 适合脚本、自动化、数据处理、Web 开发、AI 和 Agent 开发

你现在学习 Python，不是为了背很多语法，而是为了能用它完成实际任务。

对于 Agent 开发来说，Python 常用于这些事情：

- 调用大模型 API
- 组织消息列表和上下文
- 处理 JSON 数据
- 调用工具、执行流程控制
- 读取配置、日志、文件

所以你不一定要成为 Python 专家，才可以开始学 Agent；但你至少要有“能读、能改、能跑、能调试”的基础。

## 2. 解释器、脚本、编辑器，分别是什么？

很多初学者会把这几个概念混在一起，我们先把它们分开。

### 2.1 什么是解释器？

Python 解释器可以理解成“执行 Python 代码的程序”。

你写的 `.py` 文件本身不会自己运行，需要交给 Python 解释器来执行。

比如这条命令：

```bash
python hello.py
```

它的意思是：

- 用 `python` 这个解释器
- 去执行 `hello.py` 这个脚本文件

### 2.2 什么是脚本？

脚本就是你写的 Python 源代码文件，通常后缀是 `.py`。

例如：

```text
hello.py
main.py
app.py
```

这些文件里写的是你自己的代码。

### 2.3 什么是编辑器？

编辑器是你写代码的工具，比如：

- VS Code
- PyCharm
- Cursor
- 其他文本编辑器

编辑器负责帮你写代码，不负责真正执行 Python 代码。真正执行代码的是 Python 解释器。

你可以把三者理解成：

- 编辑器：写代码的地方
- 脚本：你写出来的代码文件
- 解释器：运行代码的程序

## 3. 如何确认电脑里已经安装 Python？

你可以打开终端，输入：

```bash
python --version
```

或者：

```bash
python -V
```

如果安装正常，你会看到类似输出：

```text
Python 3.11.8
```

如果没有安装成功，常见情况有：

- 提示找不到 `python`
- 提示命令不存在
- 打开的是别的程序而不是 Python

如果你后面真的要做 Agent 开发，建议使用 Python 3.10 及以上版本，通常会更省心。

## 5. 什么是交互式运行，什么是脚本运行？

Python 有两种常见使用方式。

### 5.1 交互式运行

在终端输入：

```bash
python
```

进入后你会看到类似：

```text
>>>
```

这时可以直接输入：

```python
print("hi")
```

按回车后立刻看到结果。

这种方式适合：

- 临时测试一小段代码
- 快速验证语法
- 做简单实验

### 5.2 脚本运行

把代码写进 `.py` 文件里，再执行：

```bash
python hello.py
```

这种方式适合：

- 写正式代码
- 保存程序
- 多次运行
- 做项目开发

你后面学习 Agent，主要用的是脚本运行。

## 6. Python 为什么对缩进敏感？

Python 和很多语言不一样，它不是主要靠大括号 `{}` 区分代码块，而是靠缩进。

例如：

```python
if True:
    print("yes")
```

这里 `print("yes")` 前面的缩进，表示它属于 `if` 代码块。

如果你乱写缩进，就可能报错，例如：

```python
if True:
print("yes")
```

这会导致缩进错误，因为 Python 期望 `if` 下面的代码向内缩进。

### 为什么这件事很重要？

因为你后面会经常写：

- `if`
- `for`
- `while`
- `def`
- `class`
- `try`

这些结构都依赖缩进来表示层级关系。

### 一个基本规则

通常统一使用 `4` 个空格作为一级缩进。

不要一会儿用空格，一会儿用 Tab，混用时很容易出问题。

## 7. 注释是什么？

注释是写给人看的说明文字，不会被 Python 执行。

单行注释写法：

```python
# 这是一个注释
print("hello")
```

注释适合用来：

- 解释代码用途
- 记录提醒
- 帮助自己以后回看时更快理解

但也不要把每一行都写成废话式注释。

例如这种注释就没有意义：

```python
# 打印 hello
print("hello")
```

更好的原则是：只在代码本身不够直观时再补充说明。

## 8. 一个稍微完整一点的小例子

下面写一个最简单的 Python 文件：

```python
# 这是我们的第一个 Python 小程序

name = "Python"
print("Hello")
print(name)
```

这里你先只要看懂：

- `name = "Python"` 是把一个值保存到变量里
- `print("Hello")` 会输出 `Hello`
- `print(name)` 会输出变量里的内容

运行后大致会得到：

```text
Hello
Python
```

## 9. 学 Python 时常见的几个误区

### 误区 1：一定要把语法全背下来才能开始

不是。更有效的方式是：

- 先理解基本概念
- 会查文档
- 会看报错
- 边做边学

### 误区 2：一开始就想学很复杂的框架

如果基础没恢复，直接上框架会同时卡在：

- Python 语法
- 框架概念
- 调试问题

学习体验会很差。

### 误区 3：能看懂不代表会写

看懂示例代码只是第一步。你必须自己动手：

- 新建文件
- 输入代码
- 运行
- 改错

这样手感才会回来。
